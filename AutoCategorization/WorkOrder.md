# Work Order Ticket Categorization Guidelines

**Categorize as "Work Order" if a ticket involves any of the following:**

- **User Onboarding and Account Changes:**  
  - Requests to create, modify, or disable user accounts, or prepare access for new or transitioning users.  
  - Examples: new hire onboarding, creating a user account, disabling a departing employee, setting up user permissions.

- **Software Installations and Updates:**  
  - Requests to install, upgrade, or configure approved software applications.  
  - Examples: installing business applications, updating line-of-business software, deploying standard tools.

- **Access Requests and Remote Access Setup:**  
  - Requests to grant or configure access to systems, networks, or remote environments.  
  - Examples: access to secure folders, VPN setup, remote desktop access.

- **Password Resets and Account Unlocks:**  
  - Requests to restore access when a user cannot log in due to forgotten or expired credentials.  
  - Examples: password resets, account unlocks, login assistance.

- **Hardware Requests and Recommendations:**  
  - Requests for new equipment, upgrades, or guidance on hardware purchases.  
  - Examples: requesting additional monitors, asking for printer recommendations, ordering new devices.

- **Infrastructure or Service Upgrades:**  
  - Requests to enhance or modify existing services without resolving a fault.  
  - Examples: upgrading internet service, increasing system capacity, improving connectivity options.

- **Data Restoration Requests:**  
  - Requests to recover deleted or missing files when no active system issue is present.  
  - Examples: restoring a deleted document or retrieving archived data.

- **General Assistance and “How-To” Requests:**  
  - Requests for guidance, setup help, or non-issue-related support.  
  - Examples: help using a feature, assistance configuring settings, general IT questions.

- **Autoelevate Authorization Requests:** 
  - Requests for UAC Authorization coming from Autoelevate  
  - Any of these requests should have the item set to "Autoelevate"


**A Work Order is NOT:**

- Any issue involving a system failure, error, or malfunction  
- Performance problems, outages, or degraded service  
- Troubleshooting broken hardware, software, or connectivity  
- Security incidents or suspicious activity  
- Incidents where something previously working is now not functioning  

**Examples:**

- “Access to secure network folder”  
- “New camera installation”  
- “Update line of business application”  
- “New computer installation”  
- “Need to reset my password”  

**Note:**  
Categorize as **Work Order** when the user is asking for something new, a change, access, or assistance—and nothing is broken. The defining trait is that the request is proactive or additive, not reactive to a failure.
